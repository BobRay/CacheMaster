Overview

On a large site, MODX's default behavior of clearing the site cache whenever you save a Resource can really slow things down for your visitors. The CacheMaster Plugin for MODX Revolution clears the cache for a single Resource without clearing the rest of the site cache.

Installing CacheMaster

Go to System | Package Management on the main menu in the MODX Manager and click on the "Download Extras" button. That will take you to the Revolution Repository (AKA Web Transport Facility). Put CacheMaster in the search box and press Enter. Click on the "Download" button, and when it changes to "Downloaded," click on the "Finish" button. That should bring you back to your Package Management grid. Click on the "Install" button next to CacheMaster in the grid. The CacheMaster package should now be installed.

Using CacheMaster

CacheMaster only executes when you update an existing Resource. It will not do anything at all when you create a new Resource (i.e., MODX will clear the cache normally when you save a newly created Resource).

When CacheMaster does execute, it clears the cache for the Resource you are saving, then tells MODX not to clear the site cache. CacheMaster has two modes, based on its only property: &executeAlways. The default value of the property is No. With that setting, CacheMaster will only execute when you manually uncheck the "Empty Cache" checkbox on the Settings tab before saving the Resource. The "Empty Cache" checkbox is always checked by default in MODX, so whenever you don't want the site cache cleared, you need to remember to uncheck it before saving the Resource.

If &executeAlways is set to Yes, CacheMaster will always execute regardless of the state of the "Empty Cache" checkbox — the cache for the resource will be cleared and the site cache will not. In that case, if you do want the site cache cleared, you need to clear it yourself using the Site | Clear Cache option on the Manger's Top Menu.

To set the &executeAlways property, edit the Plugin, and set the property on the Properties Tab. Don't forget to save the Plugin. Note that if you change the value to Yes it will be set back to No if you upgrade CacheMaster. To avoid that, create a property set for the Plugin.

[[$CreatePluginPropertySet? &pluginName=`CacheMaster`]]

Things to Remember

Failing to clear the site cache will speed up page loads for your site, but if you're not careful, it can leave you with pages that are not up-to-date. If, for example, you change the Menu Title of a page, any cached Wayfinder menus will show the old name until the site cache is cleared. When in doubt, you can always clear the site cache Manually using the Manager's Top Menu.

CacheMaster will clear the cache for a single Resource if you use the MODX Revolution default cache setup. It is designed to work with other caching strategies, but has not been tested with them.
