Overview

On a large site, MODX's default behavior of clearing the entire cache for all Resources and Elements whenever you save one can really slow things down for your visitors. The CacheMaster Plugin for MODX Revolution allows you to have the cache cleared for a single Resource or Element when you save it, without clearing anything else.

MODX Revolution 2.2.6 will clear the entire cache when you save a Template, regardless of whether the "Empty Cache" checkbox is checked or not and regardless of whether CacheMaster is installed. This may change in future versions of MODX, but really, it doesn't make much sense not to clear the site cache when you change a Template since any cached Resources would be out of date and would show the old layout.

Installing CacheMaster

Go to System | Package Management on the main menu in the MODX Manager and click on the "Download Extras" button. That will take you to the Revolution Repository (AKA Web Transport Facility). Put CacheMaster in the search box and press Enter. Click on the "Download" button, and when it changes to "Downloaded," click on the "Finish" button. That should bring you back to your Package Management grid. Click on the "Install" button next to CacheMaster in the grid. The CacheMaster package should now be installed.

Using CacheMaster

The default behavior of MODX is to refresh the entire site cache for all Resources and elements if the "Empty Cache" checkbox is checked and for no Resources (not even the current one) if the checkbox is not checked. Cachemaster, if the Plugin is enabled, will always clear the cache for the current object regardless of whether the "Empty Cache" checkbox is checked or not.

CacheMaster only executes when you update an existing object. It will not do anything at all when you create a new Resource or element (i.e., MODX will clear the cache normally when you save, unless you manually uncheck the "Empty Cache") checkbox.

CacheMaster has only one property: &uncheckEmptyCache. The default value of the property is No. With that setting, the "Empty Cache" checkbox on the Settings tab will be checked when you edit any Resource (as it is without the Plugin). If you leave the checkbox checked and save the Resource, the site cache will be cleared. If you uncheck it before saving the Resource, only the cache for the current object will be cleared.

If &uncheckEmptyCache is set to Yes, CacheMaster will clear the "Empty Cache" checkbox — when the Create/Edit Resource panel is loaded. If you leave it unchecked and save the object, the cache for the resource or element you are saving will be cleared and the site cache will not. If you do want the site cache cleared, you need to manually check the "Empty Cache" checkbox yourself.

To put that all another way, when CacheMaster is enabled, saving a Resource or Element with the "Empty Cache" checkbox checked will always clear the entire cache. Saving a Resource or element with the "Empty Cache" checkbox unchecked will always clear the cache just for the Resource or Element you are saving. The "&uncheckEmptyCache" property, if set, just unchecks the "Empty Cache" checkbox for you as the form is loaded.

To set the &uncheckEmptyCache property, edit the Plugin, and set the property on the Properties Tab. Don't forget to save the Plugin. Note that if you change the value to Yes it will be set back to No if you upgrade CacheMaster. To avoid that, create a property set for the Plugin.

[[$CreatePluginPropertySet? &pluginName=`CacheMaster`]]

Things to Remember

Failing to clear the site cache will speed up page loads for your site, but if you're not careful, it can leave you with pages that are not up-to-date. If, for example, you change the Menu Title of a page, any cached Wayfinder menus will show the old name until the site cache is cleared.

Objects in MODX are highly interdependent, and it can be difficult to guess what the effects of not clearing the cache will be. That's why the default behavior in MODX is to refresh the entire cache.

When in doubt, you can always clear the site cache Manually using the Manager's Top Menu.

CacheMaster will clear the cache for a single Resource or Element if you use the MODX Revolution default cache setup. It is designed to work with other caching strategies, but has not been tested with them.
